Seria A stationnaire :

Nous fixons μ=0 et σ=1. Nous générons 100 points de données (T=100) en utilisant ces paramètres, de sorte que chaque A_t est tiré de la même distribution :

𝐴_𝑡∼N(0,1),pour t=1,2,…,T. Comme μ et 𝜎 restent constants dans le temps, la distribution de A_t ne dépend pas de t.

Cela implique que la série est stationnaire, dans le sens où ses propriétés statistiques (moyenne, variance) ne changent pas avec le temps.

Meme question sur B :

Answer : Ici, nous permettons à la moyenne de changer à chaque instant t. Plus précisément, pour chaque t :

𝜇_𝑡=0.1×𝑡 ,𝜎_𝑡=1.

Ainsi, chaque 𝐵_𝑡 est tiré de la distribution suivante :

𝐵_𝑡 ∼N(0.1t,1).

À mesure que t augmente, la moyenne de la distribution augmente progressivement. Cela rend la série non stationnaire, car sa valeur attendue dépend du temps t.

def stationarity_test(X, cutoff=0.05):
    # H_0 in adfuller is unit root exists (non-stationary)
    # We must observe significant p-value to convince ourselves that the series is stationary
    pvalue = adfuller(X)[1]
    if pvalue < cutoff:
        print('p-value = ' + str(pvalue) + ' The series ' + X.name +' is likely stationary.')
    else:
        print('p-value = ' + str(pvalue) + ' The series ' + X.name +' is likely non-stationary.')

Conclusion :¶

Pourquoi la stationnarité est-elle importante ?

Une série temporelle stationnaire possède des propriétés statistiques constantes dans le temps (c’est-à-dire que la moyenne, la variance et la structure d’autocorrélation ne changent pas).

Une série temporelle non stationnaire, en revanche, a au moins une de ces propriétés qui évolue avec le temps (souvent la moyenne). Dans le cadre de la modélisation financière, de nombreuses séries de prix montrent des tendances ou une volatilité variable, ce qui les rend non stationnaires.

➡️ Pourquoi c’est important en finance quantitative ? Identifier la stationnarité est essentiel car de nombreux modèles statistiques et de prévision (comme les modèles ARIMA, les tests de cointégration, etc.) supposent ou nécessitent que la série étudiée soit stationnaire.

  • Avant d’appliquer un modèle, on teste la stationnarité (ex : test ADF - Augmented Dickey-Fuller)
  • Si la série n’est pas stationnaire, on peut la transformer (ex : différenciation, log-transformation, filtrage) pour qu’elle le devienne.

La stationnarité est cruciale pour la cointégration 📌 Problème :

La cointégration est une propriété qui permet d’identifier des relations de long terme entre plusieurs séries financières (ex : deux actifs fortement liés). Si deux séries sont cointegrées, cela signifie qu’il existe une relation stable entre elles et que leur écart (spread) est stationnaire.

📌 Exemple :

Supposons qu’on observe les prix de Binance et Deribit pour un même contrat futur. Individuellement, ces prix peuvent être non stationnaires (car ils suivent des tendances du marché). Mais si leur spread est stationnaire, cela signifie qu’ils évoluent de manière cohérente à long terme. Ce type d’analyse permet d’identifier des opportunités d’arbitrage et de pairs trading.

✅ Solution :

On teste la cointégration en vérifiant si une combinaison linéaire des deux séries est stationnaire (ex : test d’Engle-Granger). Si le spread est stationnaire, cela signifie qu’il y a une relation structurelle forte entre les actifs → signal exploitable en trading.

def fetch_symbol_closes_df(symbol_str):
    import ccxt, pandas as pd
    from functools import reduce

    symbol = symbol_str
    timeframe = '15m'
    start_date = '2025-01-01T00:00:00Z'
    exchanges = ['binance', 'deribit']
    frames = []

    for ex_id in exchanges:
        ex = getattr(ccxt, ex_id)()
        ex.load_markets()

        since = ex.parse8601(start_date)
        limit = 500
        all_ohlcv = []
        
        while True:
            ohlcv = ex.fetch_ohlcv(symbol, timeframe, since, limit)
            if not ohlcv:
                break
            all_ohlcv += ohlcv

            since = ohlcv[-1][0] + 1  
            if len(ohlcv) < limit:
                break

        df = pd.DataFrame(all_ohlcv, columns=['ts','o','h','l','c','v'])
        df['datetime'] = pd.to_datetime(df['ts'], unit='ms')
        df = df[['datetime','c']].rename(columns={'c': ex_id})
        frames.append(df)
    
    final_df = reduce(lambda left, right: pd.merge(left, right, on='datetime', how='outer'), frames)
    final_df.sort_values('datetime', inplace=True)
    final_df.reset_index(drop=True, inplace=True)
    return final_df

def estimate_coint_and_adf(df):

    df = df.dropna(subset=['binance', 'deribit']).copy()

    # 2. Y = binance, X = deribit + intercept
    Y = df['binance']
    X = sm.add_constant(df['deribit'])  # Ajoute l'intercept

    # 3. Régression OLS => Y = alpha + beta * X
    model = sm.OLS(Y, X).fit()

    alpha = model.params['const']
    beta  = model.params['deribit']

    # 4. Résidu (epsilon)
    df['epsilon'] = Y - alpha - beta * df['deribit']

    # 5. Test ADF sur epsilon
    adf_result = ts.adfuller(df['epsilon'], autolag='AIC')
    p_value = adf_result[1]

    return alpha, beta, df['epsilon'], p_value, model

Génération d'un signal basé sur le z-score¶

Z-Score :

zscore = (spread_smooth − mean_spread) / std_spread

Le Z-Score indique de combien d’écarts-types le spread courant s’éloigne de sa moyenne.

  • Un Z-Score proche de 0 signifie que le spread est “normal” (autour de sa moyenne).
  • Un Z-Score élevé (supérieur à +2, par exemple) signifie que le spread est “trop haut” par rapport à sa moyenne historique.
  • Un Z-Score bas (inférieur à –2) signifie que le spread est “trop bas”.

window = 500

df['mean_spread'] = df['spread_smooth'].rolling(window).mean()
df['std_spread']  = df['spread_smooth'].rolling(window).std()

# Calcule le Z-score du spread
df['zscore'] = (df['spread_smooth'] - df['mean_spread']) / df['std_spread']

# Exemple simple de stratégie
# Si zscore > +2 => spread "trop haut" => short le spread (short binance, long deribit)
# Si zscore < -2 => spread "trop bas"  => long le spread  (long binance, short deribit)

df['signal'] = 0
df.loc[df['zscore'] >  2, 'signal'] = -1  # short spread
df.loc[df['zscore'] < -2, 'signal'] =  1  # long spread

# Optionnel: fermer la position si le zscore repasse autour de 0
df.loc[df['zscore'].abs() < 0.5, 'signal'] = 0

Signaux de trading :

  • Quand le Z-Score dépasse +2, on considère le spread surévalué → on prend une position short sur le spread.

(Dans le cas d’un spread = prix Binance – prix Deribit, être short sur le spread reviendrait à vendre Binance et acheter Deribit.)

  • Quand le Z-Score passe en dessous de –2, on considère le spread sous-évalué → on prend une position long sur le spread.

(Acheter Binance et vendre Deribit, par exemple.)

  • Pour éviter de garder des positions ouvertes quand le signal redevient neutre, on fixe une zone de “sortie” ou de “non-position” lorsque le Z-Score se rapproche de 0 (par exemple si abs(zscore) < 0.5).

def plot_trades(df, price_col='spread', signal_col='signal'):
    
    # On va stocker les points (x,y) pour chaque événement
    enter_long_x, enter_long_y = [], []
    enter_short_x, enter_short_y = [], []
    exit_long_x, exit_long_y = [], []
    exit_short_x, exit_short_y = [], []
    
    # Pour parcourir les changements de signal
    df = df.copy().reset_index(drop=True)
    
    # Signal initial
    prev_sig = df.loc[df.index[0], signal_col]
    
    # Boucle sur chaque rangée à partir de la 2ème
    for i in range(1, len(df)):
        cur_sig = df.loc[i, signal_col]

        if cur_sig != prev_sig:
            # => Le signal vient de changer
            if cur_sig == 1:  
                # Changement vers un long
                enter_long_x.append(df.loc[i, 'datetime'])
                enter_long_y.append(df.loc[i, price_col])
            elif cur_sig == -1:
                # Changement vers un short
                enter_short_x.append(df.loc[i, 'datetime'])
                enter_short_y.append(df.loc[i, price_col])
            else:
                # cur_sig == 0 => On sort d'une position
                if prev_sig == 1:
                    # On sort d'un long
                    exit_long_x.append(df.loc[i, 'datetime'])
                    exit_long_y.append(df.loc[i, price_col])
                elif prev_sig == -1:
                    # On sort d'un short
                    exit_short_x.append(df.loc[i, 'datetime'])
                    exit_short_y.append(df.loc[i, price_col])
        
        prev_sig = cur_sig
    
    # --- Tracé du graphique ---
    plt.figure(figsize=(12, 6))
    plt.plot(df['datetime'], df[price_col], label=price_col, color='black')

    # Entrées en position
    plt.scatter(enter_long_x, enter_long_y, marker='^', color='green', s=100, label='Enter LONG')
    plt.scatter(enter_short_x, enter_short_y, marker='v', color='green', s=100, label='Enter SHORT')
    # Sorties de position
    plt.scatter(exit_long_x, exit_long_y, marker='v', color='red', s=100, label='Exit LONG')
    plt.scatter(exit_short_x, exit_short_y, marker='^', color='red', s=100, label='Exit SHORT')
    
    plt.title("Backtest avec entrées/sorties de position")
    plt.legend()
    plt.show()

plot_trades(df, price_col='spread_smooth', signal_col='signal')
